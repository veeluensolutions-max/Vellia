Edita arquivos com precisão cirúrgica. Pode fazer substituições pontuais ou criar arquivos inteiros do zero, mantendo o contexto do projeto.

Ferramentas

Ferramentas poderosas
ao seu comando.

Conheça todas as ferramentas disponíveis e como usá-las por linguagem natural.

📁 Arquivos

write_to_file Criar

Cria novos arquivos com conteúdo completo

Exemplo: "Crie um arquivo utils.ts com funções de formatação"
replace_file_content Editar

Edita trechos específicos de um arquivo existente

Exemplo: "Mude a cor do botão de azul para roxo"
view_file Ler

Lê o conteúdo de arquivos de texto e imagens

Exemplo: "Mostre o conteúdo do arquivo package.json"
list_dir Navegar

Lista os arquivos e pastas de um diretório

Exemplo: "O que tem na pasta src?"
grep_search Buscar

Busca padrões de texto dentro dos arquivos do projeto

Exemplo: "Onde usamos a função handleSubmit?"

⚡ Execução

run_command Terminal

Executa comandos no terminal do seu sistema

Exemplo: "Instale o pacote axios e rode os testes"
browser_subagent Navegador

Controla um navegador para interagir com páginas web

Exemplo: "Abra o site e faça login com essas credenciais"
schedule Timer

Agenda timers e tarefas recorrentes (cron)

Exemplo: "A cada 5 minutos verifique o status do deploy"

🌐 Web & Pesquisa

search_web Pesquisar

Faz buscas na web e retorna resultados resumidos

Exemplo: "Pesquise como configurar Prisma com PostgreSQL"
read_url_content URL

Lê o conteúdo de uma URL e converte para texto

Exemplo: "Leia a documentação em https://docs.example.com"
generate_image Imagem

Gera imagens com IA a partir de descrições em texto

Exemplo: "Gere um ícone minimalista de um foguete"
Slash Commands

Atalhos poderosos
na ponta dos dedos.

Digite esses comandos no chat para ativar workflows especializados.

/goal

Modo Objetivo

Para tarefas longas e complexas. O agente trabalha de forma contínua até atingir o objetivo completo, sem parar no meio.

Ideal para: Refatorações grandes, migrações, tarefas overnight
/schedule

Agendar Tarefas

Configura tarefas recorrentes ou timers únicos. Perfeito para monitoramento e automações periódicas.

Ideal para: Monitorar deploys, health checks, verificações periódicas
/grill-me

Entrevista Interativa

O agente faz perguntas para alinhar decisões de design e requisitos antes de começar a implementar.

Ideal para: Novos projetos, decisões de arquitetura, requisitos vagos
/learn

Aprender Padrões

Ensina o agente a lembrar de padrões, convenções ou configurações específicas do seu projeto.

Ideal para: Após corrigir o agente, setups complexos, convenções do time
Dicas Pro

Extraia o máximo
do Antigravity.

Técnicas para tornar suas interações mais eficientes e resultados mais precisos.

01

Seja Específico

Quanto mais detalhes você fornecer, melhor o resultado. Em vez de "faça um site", diga "crie uma landing page com hero section, dark mode, e seção de preços com 3 planos".

❌ Vago

"Faça um formulário"

✅ Específico

"Crie um formulário de cadastro com campos nome, email e senha, com validação e feedback visual"

02

Itere Incrementalmente

Não tente fazer tudo de uma vez. Comece com a estrutura básica, depois peça melhorias pontuais como "adicione animações", "melhore as cores", "torne responsivo".

03

Use Referências

Aponte para arquivos existentes como referência: "Siga o mesmo padrão do componente Header.tsx" ou "Use o mesmo estilo de animação da página de login".

04

Aproveite o Contexto

O Antigravity vê quais arquivos você tem abertos no editor. Abra os arquivos relevantes antes de pedir alterações para dar mais contexto.

05

Revise os Planos

Para tarefas complexas, o agente cria um plano de implementação. Revise com atenção antes de aprovar — é mais fácil corrigir no plano do que no código.

06

Ensine com /learn

Quando corrigir o agente, use /learn para que ele memorize a preferência. Na próxima vez, ele já saberá fazer do jeito certo.
